产品概述

什么是 Jina Reader？

Jina Reader 是由 Jina AI 开发的一款 Web 内容提取工具，能够将任意网页 URL 转换为干净、结构化、LLM 友好的格式（Markdown / JSON / 纯文本）。它的核心理念极其简洁：只需在目标 URL 前添加 https://r.jina.ai/ 前缀，即可获得去除广告、导航栏、页脚等噪音的高质量主要内容。

一句话总结：Jina Reader = 为 LLM 量身打造的智能网页爬虫，零配置、开箱即用，是 RAG 系统和 AI Agent 获取实时网页信息的核心基础设施。

核心功能与技术架构

技术原理

Jina Reader 的处理流程分为三个关键步骤：

  • 无头浏览器渲染 — 使用 Headless Browser 加载目标页面，确保动态内容（JavaScript 渲染）也能被正确抓取
  • 主要内容提取 — 基于 Mozilla Readability 算法，智能识别并提取页面主体内容，自动去除导航栏、侧边栏、页脚、广告等噪音元素
  • 格式转换与清洗 — 清除冗余的 HTML 标签、JavaScript 代码和 CSS 样式，将内容转换为 Markdown / JSON / 纯文本等 LLM 友好格式

内容提取能力

  • 自动去除广告、导航、页脚
  • 保留标题、段落、列表等结构
  • 原生支持 PDF 文件读取
  • 图片自动生成 alt 描述

多种输出格式

  • markdown — 默认格式，LLM 最友好
  • html — 清洗后的 HTML
  • text — 纯文本
  • screenshot — 页面截图

结构化数据提取

  • 通过 x-json-schema Header 定义 JSON Schema
  • 通过 x-instruction Header 使用自然语言指令
  • 自动提取价格、日期、标题等字段

高级特性

  • 流式处理，适合大型动态网页
  • 多语言支持（29 种语言）
  • 代理模式绕过反爬机制
  • 平均响应时间 < 2 秒

ReaderLM 模型

ReaderLM-v2：专为 HTML→Markdown 设计的小语言模型

ReaderLM-v2 是 Jina AI 发布的 1.5B 参数小语言模型（SLM），专门针对 HTML 到 Markdown/JSON 的转换任务进行了优化训练。它已完全集成到 Reader API 中，为结构化数据提取提供支撑。

  • 相比 v1 版本，质量提升 3 倍
  • 支持更长的上下文窗口
  • 覆盖 29 种语言
  • 可通过 JSON Schema 或自然语言指令提取特定字段
  • 模型已在 Hugging Face 上开源

使用方式

最简用法：URL 前缀

无需任何 API Key，直接在浏览器或代码中使用：

https://r.jina.ai/https://example.com

API 调用（带 Key）

curl -H "Authorization: Bearer jina_xxxx" \ -H "X-Respond-With: markdown" \ https://r.jina.ai/https://example.com

结构化数据提取

curl -H "Authorization: Bearer jina_xxxx" \ -H 'X-Json-Schema: {"type":"object","properties":{"title":{"type":"string"},"price":{"type":"number"}}}' \ https://r.jina.ai/https://example.com/product

搜索模式

除了读取 URL，还支持搜索模式：

https://s.jina.ai/what+is+jina+reader

推荐策略：使用 Jina Reader 做快速原型验证（零成本启动），需要全站爬取和生产级稳定性时切换到 Firecrawl。两者并非互斥，可以在同一项目中互补使用。

应用场景

RAG 系统

为检索增强生成（RAG）管道提供干净的网页内容源。将实时网页转为 LLM 可直接消费的 Markdown，大幅降低预处理成本。

AI Agent 网页浏览

作为 AI Agent 的"眼睛"，让 Agent 能够实时读取和理解任意网页内容，是 MCP Server 和 Tool Use 的理想搭配。

信息提取与监控

结合 JSON Schema 提取功能，可自动从产品页、新闻站点等提取结构化数据，用于价格监控、竞品分析等。

知识库构建

批量将网页文档转换为 Markdown 格式，导入 Notion、Obsidian 等知识管理系统，或用于向量数据库的文档预处理。

总结与建议

优势

  • 极低门槛 — URL 前缀即可使用，无需任何配置和注册
  • 免费额度充裕 — 1000 万 Token 足够原型和小规模使用
  • 自有模型加持 — ReaderLM-v2 提供高质量 HTML→Markdown 转换
  • Apache-2.0 开源 — 企业友好的许可协议，可自部署
  • Elastic 背书 — 被 Elastic 收购后，长期维护和发展有保障

局限

  • 仅支持单页提取 — 不具备站点级爬取和自动翻页能力
  • Token 计费不可预测 — 费用随页面内容长度变化，大规模使用时成本难以精确估算
  • 无浏览器自动化 — 无法处理需要登录、点击交互的页面

适用人群：如果你正在构建 RAG 应用、AI Agent 或需要将网页内容喂给 LLM，Jina Reader 是目前最简便且性价比最高的选择之一。对于需要全站爬取或复杂交互的场景，建议搭配 Firecrawl 或 ScrapeGraphAI 使用。
